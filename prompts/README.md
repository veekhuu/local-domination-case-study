# Reconstructed Prompts

These prompts are reconstructed from memory and the case study write-up, not copy-pasted from actual sessions. The original workflow used **Wispr Flow** (voice-to-text) to speak naturally to Claude, so there were never "typed prompts" to save. What you see here represents the intent and approach, adapted into usable prompts you can customize for your own projects.

## How to Use These Prompts

Follow these phases in order. Each phase builds on the output of the previous one.

1. **Planning** ([planning.md](./planning.md)) -- Claude or any LLM
2. **Scrollytelling** ([scrollytelling.md](./scrollytelling.md)) -- Gemini, ChatGPT Custom GPTs, fal.ai
3. **Intelligence** ([intelligence.md](./intelligence.md)) -- Manus AI, Claude + Tavily/Perplexity
4. **Content** ([content.md](./content.md)) -- Claude Code
5. **Technical SEO** ([technical-seo.md](./technical-seo.md)) -- Claude Code

## Customization

Throughout these prompts you will see `[BRACKETED PLACEHOLDERS]` like `[COMPANY TYPE]`, `[TARGET MARKET]`, and `[LIST OF SERVICES]`. Replace every bracketed placeholder with your own business details before using the prompt. The prompts are written generically so they apply to any local service business, but they work best when you fill in specifics.

## Important Notes

- **Order matters.** Intelligence gathering (Phase 3) feeds directly into content creation (Phase 4). Skipping ahead or doing phases out of order means your content will lack the competitive and market data that makes it effective.
- **Adapt to your market.** These prompts were built around a home construction company in a specific metro area. Your industry, competitors, and local market are different. Rewrite the details, not just the placeholders.
- **Tool alternatives exist.** Some prompts reference specific tools (fal.ai, Manus AI, Wispr Flow, etc.). If you do not have access to a particular tool, there are alternatives. The prompt logic transfers -- only the interface changes.
